A CLI mail client  ·  v0.5.0  ·  Node ≥ 20

mmmail.

Stateless commands — list, read, send, reply, search — for Gmail, any IMAP server, or any SMTP host. No GUI, no inbox to manage. Just mail, where you already work.

$ npm install -g @0xmmo/mmmail
§ 01
Why it exists

Email is just text,
so why does it need a browser?

// stateless

No daemon, no local sync. Every command is a fresh round-trip — list, read, send, exit. Nothing to babysit.

// keychain-first

Passwords, OAuth secrets, refresh tokens — all stored in your OS keychain. They never touch the config file.

// pipeable

Read into less. Pipe stdin into a draft. Compose in $EDITOR. The CLI is the interface.

// multi-account

Gmail, Outlook, Fastmail, iCloud, Yahoo, plus any IMAP/SMTP host. Switch with -a.

// own your client

Your own Google OAuth client, in testing mode. No verification dance, no shared dependency, no rate-limit roulette.

// MIT, ~zero deps

Small, readable surface. imapflow, nodemailer, commander. That's most of it.

§ 02
Quickstart

Five minutes from install to send.

# add an account — Gmail (OAuth) or any IMAP host
$ mmm init

# the 20 most recent messages in INBOX
$ mmm list

# open a single message by UID
$ mmm read 1234

# send (body via -b, '-' for stdin, omit for $EDITOR)
$ mmm send -t a@example.com -s "hi" -b "hello"

# attach files (repeatable; works on send and reply)
$ mmm send -t a@example.com -s "report" -b "see attached" --attach ./report.pdf

# reply (or reply-all with --all)
$ mmm reply 1234 -b "thanks"

# save attachments from a received message to disk
$ mmm read 1234 --save-attachments ./out

# full-text search across the mailbox
$ mmm search "invoice"

§ 04
Account setup

Get connected in five minutes.

Gmail · OAuth (one-time per machine, ~3 min)

Gmail's https://mail.google.com/ scope is restricted, so a shared OAuth client isn't possible. You'll create your own desktop OAuth client — mmm init walks you through it. Stays in testing mode (no verification dance), up to 100 test users.

  1. Create a Google Cloud project — console.cloud.google.com/projectcreate
  2. Enable the Gmail API — console.cloud.google.com/apis/library/gmail
  3. Configure the OAuth consent screen (Audience: External) — console.cloud.google.com/auth/overview
  4. Add yourself as a test user — console.cloud.google.com/auth/audience
  5. Create OAuth credentials of type Desktop app — console.cloud.google.com/auth/clients
  6. Copy the client ID + secret (or download the JSON), then run:
mmm add google --email me@gmail.com --credentials-file ./client_secret.json

Subsequent Gmail accounts on the same machine skip the project setup and only run the consent flow.

Microsoft · OAuth (zero Azure setup)

mmm ships with a built-in Microsoft Entra app (multi-tenant + personal accounts, public client + PKCE, no secret). Just run:

mmm add microsoft --email me@outlook.com

You'll see "mmmail" on the consent screen, approve, and you're done.

Work/school accounts: many M365 tenants disable SMTP AUTH by default. If mmm send fails with SmtpClientAuthentication is disabled, ask your admin to enable authenticated client SMTP submission for your mailbox.

Prefer your own Entra app? Register one (Account type: Accounts in any organizational directory and personal Microsoft accounts), add http://localhost as a Mobile and desktop applications redirect URI, enable Allow public client flows, then pass your client ID:

mmm add microsoft --email me@outlook.com --client-id <your-id>
IMAP / SMTP · App password

Fastmail, iCloud, Yahoo, and any standard IMAP host work via app password — stored in your OS keychain, never in the config file.

mmm add imap --email me@example.com --preset fastmail --password-stdin

Without a preset, pass --imap-host, --imap-port, --smtp-host, --smtp-port explicitly. Use --smtp-password-stdin if your provider requires a different password for SMTP.

§ 05
Command reference

Every command, in one place.

// reading

mmm list
20 most recent in a folder. Flags: -f <name>, -n <limit>, -u (unread only), --since, --before, --uid-after.
mmm read <uid>
Open one message. -f <folder>, --include-html (large), --save-attachments <dir>.
mmm search <q>
Full-text across subject/body/from. Same time/pagination flags as list.
mmm folders
List available IMAP folders for the active account.

// writing

mmm send
-t <addr...> -s <subj>. Body: -b, --body-stdin, or omit for $EDITOR. -c/-B for cc/bcc, --attach <path...> for files.
mmm reply <uid>
Auto-threads (In-Reply-To, References). --all for reply-all, --attach for files.

// mailbox actions

mmm mark <uid>
--read / --unread, --flagged / --unflagged.
mmm move <uid> <dest>
Move to another folder, e.g. "[Gmail]/All Mail".
mmm delete <uid>
Permanent — sets \Deleted and EXPUNGEs.

// accounts & health

mmm
Interactive dashboard (TTY); non-TTY prints help and exits 2.
mmm accounts
List configured accounts. mmm default <email> sets the active one, mmm remove drops it.
mmm add <provider>
Non-interactive: google, microsoft, imap. See § 04 setup.
mmm doctor
Probe every account; exits 1 on any failure — easy CI gate.

Every command takes -a, --account <email> to override the default account, and --json for machine-readable output.

§ 06
Attachments

Send files. Save them out.

Attach files to outgoing mail with --attach (repeatable). On the way in, list attachments in the message view automatically; pull the bytes to disk with --save-attachments. 

# attach one or many files (works on send and reply)
$ mmm send -t a@example.com -s "Q3 numbers" -b "see attached" \
    --attach ./report.pdf --attach ./spreadsheet.xlsx

$ mmm reply 1234 --attach ./signed.pdf

# attachments appear in the rendered message
$ mmm read 1234
Subject: Q3 numbers
From:    a@example.com
Attachments:
  1. report.pdf (application/pdf, 124.2 KB)
  2. spreadsheet.xlsx (application/vnd.openxmlformats-..., 38.4 KB)

# save them to disk — collisions get -1, -2, ... suffixes
$ mmm read 1234 --save-attachments ./out
✓ saved /Users/you/out/report.pdf
✓ saved /Users/you/out/spreadsheet.xlsx

# JSON output for agents
$ mmm read 1234 --save-attachments ./out --json | jq .attachments

Filenames are sanitised (path separators stripped, leading dots removed); empty/invalid names fall back to attachment-N. --json includes filename, contentType, size, contentId, inline, and (when saved) savedPath.

§ 07
For agents & scripts

Built to be piped.

// --json everywhere

Every data command supports --json — list, read, search, send, reply, mark, move, delete, folders, doctor, accounts.

// fail-fast bare invocation

mmm with no subcommand in a non-TTY exits status 2, so scripts that forgot a command fail loudly instead of hanging on a dashboard.

// health gate

mmm doctor --json exits 1 if any account fails to connect — drop it into CI as a tripwire.

// secrets off argv

--body-stdin, --password-stdin, --client-secret-stdin keep secrets out of argv and shell history.

// pagination

--uid-after <n> for incremental sync, --since / --before for time windows. Works on list and search.

// HTML off by default

read --json drops the HTML body (often 50× the text). Pass --include-html when you need it.
